NanoBot.net P6

智能体项目横向调研与吸收路线 · 2026-06-08

P6 是什么

P6 是 NanoBot.net 在继续施工新功能之前的一次横向调研阶段。核心原则是： 一个问题不能两头都是谜团。先把外部智能体项目研究清楚，再决定 NanoBot.net 后续吸收什么。

本轮共调研 7 个外部智能体项目，研究副本已浅克隆到 C:\Users\Administrator\Documents\Github\_agent-research-p6， 分册版 Wiki 入口见 docs/p6-wiki/00-index.zh-CN.md。

总体结论：NanoBot.net 仍然最适合作为主线。
这 7 个项目各有价值，但没有一个同时满足 NanoBot 的目标组合 —— .NET 8 主线、Apache-2.0、 本地优先、不用 WebView2/Electron 做桌面壳、能分发 MSI、能和 Nong/GroundPA-Toolkit 走 plugin/bootstrap、 能把 DeepSeek V4 Flash / GitCode / DMX / Nong / WebUI / 未来原生 UI 统一到一个 runtime。

调研的 7 个项目

1. CodeWhale.net

Rust 最像 runtime

DeepSeek-TUI 改名后的 Rust 终端 coding agent，围绕 DeepSeek V4 Pro/Flash 构建。

对 NanoBot 最有价值： DeepSeek V4 Flash profile、reasoning_content 回传、cache hit/miss 成本统计、 子 agent 后台并发模型、LSP post-edit diagnostics、side-git snapshot、durable task manager、 approval policy 与工具级 ask rule。
风险：Rust 实现不能直接移植，部分 sandbox 能力 Windows 不可等价。

2. DeepSeek-GUI.net

Electron/Vite UI 参考

以 Kun 为本地运行时的桌面工作台，产品形态完整：Code、Write、手机连接、Skill/MCP 管理、计划、Review。

对 NanoBot 最有价值： WebUI 信息架构、HTTP/SSE 边界、ImmutablePrefix 稳定指纹、 token economy 设置、FileSessionStore + JSONL event replay、approvalGate/userInputGate 模型。
风险：Electron 桌面壳不符合要求，Kun 是 TypeScript runtime。

3. PilotDeck.net

AGPL-3.0 Agent OS 参考

WorkSpace-centric Agent OS：workspace 隔离、白盒 memory、smart routing、always-on 后台执行。

对 NanoBot 最有价值： WorkSpace 一等边界、白盒 memory 可追踪/可编辑/可回滚、 Smart routing 按任务难度分派模型、Always-on 任务生命周期、 plugin.json 扩展协议、多渠道 SessionMapper。
风险：AGPL-3.0，Node/TS 体系，不适合直接吸收代码。

4. GenericAgent.net

Python 极简工具参考

极简、自进化、自举型 Python agent。9 个原子工具 + ~100 行 agent loop + 分层记忆 + 自动沉淀 skill。

对 NanoBot 最有价值： 工具集最小化、自进化 skill 沉淀、L0-L4 分层 memory、 Morphling mode 外部能力吸收、Agent BBS 多 agent 协作。
风险：code_run 权限过大，NanoBot 须保留严格 allowlist。

5. EvoScientist.net

Python 多 agent 参考

科研工作流 agent，基于 DeepAgents/LangGraph，多 agent team + 科研 lifecycle + 多渠道。

对 NanoBot 最有价值： 多渠道 message bus 统一调度、channel session key 边界、 streaming event 分类（thinking/tool_call/subagent_text/approval/ask_user）、 科研 workflow 角色拆分、Agent 主动问用户。
风险：Python/LangGraph 路线与 .NET 主线差异大。

6. soloncode.net

Java/Solon 三端体验参考

Java coding agent，CLI/Web/IDE 三端统一，中文提示驱动，Solon ReActAgent 扩展。

对 NanoBot 最有价值： 一个 runtime 同时服务 CLI/Web/IDE、@agentName task 交互、 extension demo builder/interceptor、中文优先配置体验、 channel 绑定与本地 credential store。
风险：Java/Solon 体系，sessionWindowSize=8 与 1M 长上下文路线不同。

7. agent-framework.net

.NET 抽象层参考

Microsoft 生产级 .NET/Python agent 和 workflow 框架，不是 coding agent 产品而是基础设施。

对 NanoBot 最有价值： .NET agent/workflow abstraction、Human-in-the-loop 节点、 OpenTelemetry tracing、Durable workflow/checkpointing、 Declarative agents/skills schema、ASP.NET + OpenAI-compatible hosting。
风险：体系很重，容易把 NanoBot 变成框架集成项目。

不适合作为主线的项目

  • DeepSeek-GUI.net：Electron 桌面壳，不符合用户桌面分发偏好。
  • CodeWhale.net：Rust 主线太强，适合参考，不适合替换 .NET 主线。
  • PilotDeck.net：AGPL-3.0，协议和技术栈都不适合直接吸收代码。
  • GenericAgent.net：Python 极简路线优秀，但和 .NET/Nong/GroundPA 统一目标不一致。
  • EvoScientist.net：科研 agent 方向强，但主线偏 Python + LangGraph/DeepAgents。
  • soloncode.net：Java/Solon 生态，适合作为三端和中文体验参考。
  • agent-framework.net：可学抽象，不能把 NanoBot 变成 Azure/Foundry 重框架附属。

P6 吸收策略

先吸收共识能力，不追单个项目

多个项目都重复出现的能力，优先进入 NanoBot：

  • Runtime API + SSE event stream
  • Stable session/thread/turn/item model
  • ToolRegistry + ToolRuntime + approval policy
  • Memory 可视化和可编辑
  • Workspace isolation
  • Plugin/Skill marketplace
  • DeepSeek V4 Flash reasoning/cache usage
  • 多渠道 session mapper

NanoBot 不换主线

P6 调研不是为了决定"换成某个项目"，而是为了让 NanoBot 少走弯路。每个项目的角色：

NanoBot.net          = .NET 8 GroundPA runtime host（主线）
CodeWhale / Kun      = DeepSeek 和 token economy 教材
PilotDeck            = workspace / memory / router / always-on 教材
GenericAgent         = 极简工具和自进化 skill 教材
EvoScientist         = 多 agent 科研 workflow 和 channel bus 教材
soloncode            = 中文 CLI/Web/IDE 体验教材
agent-framework      = .NET workflow / hosting / observability 教材

P6 之后建议施工顺序

  1. DeepSeek V4 Flash provider 和 usage/cache metrics模型接入与用量统计
  2. Stable context rendererimmutable prefix、工具 schema 排序、历史工具结果冻结
  3. Runtime event modelcontent / reasoning / tool / usage / cache / approval / user_input 事件
  4. Session/thread/turn/item 持久化会话结构落盘与恢复
  5. WebUI 展示reasoning、tool detail、cache hit rate、workspace、memory 可视化
  6. Plugin/Skill marketplace 设计扩展市场架构
  7. Workspace-scoped white-box memory工作区级白盒记忆
  8. Smart routing / sub-agent / always-on task智能路由、子代理、后台任务

顺序逻辑：先让 NanoBot 的单 agent loop 可靠、可观测、可持久，再上多 agent、always-on 和复杂 workspace。
