# 2026-06-13 NanoBot 应用层 Phase 10 — 端到端验证清单

## 验证用例

以下用例需要启动 NanoBot (CLI chat 模式或 WebSocket 模式) 并连接 LLM provider 后执行。

### 用例 1: Excel → Chart 管线

```
用户: "分析 data.xlsx 的 A 列处理组和 B 列产量数据，画柱状图"
预期:
  1. LLM 调用 excel_to_groups tool
  2. 收到分组 JSON 后调用 chart_bar tool
  3. 两次 tool call 均未被 ConfirmationHook 拦截 (readonly 命令)
  4. 返回 PNG 路径
```

### 用例 2: 论文诊断

```
用户: "检查这篇论文 paper.docx 的参考文献"
预期:
  1. LLM 调用 inspect_refs tool
  2. 返回诊断结果 (JSON 格式)
```

### 用例 3: PDF 合并

```
用户: "把 a.pdf 和 b.pdf 合并成 combined.pdf"
预期:
  1. LLM 调用 pdf_merge tool
  2. ConfirmationHook 拦截 (write 命令) → LLM 要求确认
  3. 用户回复 "confirm" → 重新调用 → 放行
  4. 返回合并文件路径
```

### 用例 4: OCR 环境检查

```
用户: "检查 OCR 环境状态"
预期:
  1. LLM 调用 ocr_check_env tool
  2. 返回 v5/v6 模型状态 JSON
```

### 用例 5: 确认机制 (安装模型)

```
用户: "安装 PP-OCRv6 medium 模型"
预期:
  1. LLM 调用 ocr_install_model tool
  2. ConfirmationHook 拦截 (install 命令) → 返回 "[Nong guard] ... Reply confirm to proceed"
  3. LLM 向用户说明需要确认
  4. 用户回复 "confirm" → 重新调用 → 放行
```
